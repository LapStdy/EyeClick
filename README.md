## 概述

EyeClick 是一款基于计算机视觉（OpenCV 模板匹配）的自动化鼠标点击工具。它通过识别屏幕上的图像模板来定位目标，并模拟鼠标点击操作。支持多尺度匹配、窗口绑定、搜索区域限定、点击偏移等高级功能，适用于游戏自动化、软件测试、重复操作替代等场景。

## 功能特性

### 核心功能

| 功能 | 说明 |
|------|------|
| **图像模板匹配** | 使用 OpenCV 的 `TM_CCOEFF_NORMED` 算法，支持多尺度缩放匹配 |
| **多尺度搜索** | 默认在 0.8、0.9、1.0、1.1、1.2 五种尺度下搜索，也可固定单一尺度 |
| **搜索区域限定** | 可设置矩形搜索区域，仅在指定范围内查找模板，提高匹配效率和准确性 |
| **窗口绑定** | 绑定目标窗口后，搜索区域会自动偏移到窗口客户区坐标系下 |
| **点击偏移量** | 匹配到目标后可设置水平和垂直偏移量，精确定位到模板内部的特定位置 |
| **重试机制** | 匹配失败时可自动重试，可配置重试次数和间隔时间 |
| **执行倒计时** | 每条指令执行前可设置倒计时，给予充分的准备时间 |
| **置信度阈值** | 可调节匹配置信度阈值（0~1），平衡匹配精度和成功率 |

### 任务管理

- 支持**左键点击**、**右键点击**、**等待（无操作）** 三种指令类型
- 指令列表支持添加、编辑、删除、上移、下移、清空
- 设置自动保存到 `config.json`
- 启动后自动执行选项

### 测试与调试

- **测试匹配**：在执行前对单个指令进行匹配测试，显示匹配位置、置信度和可视化标记
- 实时日志输出，记录执行过程和结果
- 状态指示器显示当前进度

### 界面特性

- 基于 tkinter 的图形界面
- 高亮显示匹配位置（截图预览中用绿色矩形和红色中心点标记）
- 模板图片预览
- 执行前自动最小化主窗口选项
- 完成后自动关闭程序选项

## 安装

### 环境要求

- Windows 7 及以上操作系统
- Python 3.8+

### 从源码运行

```bash
# 克隆或下载项目
cd EyeClick

# 安装依赖
pip install -r requirements.txt

# 运行程序
python main.py
```

### 依赖清单

```
pynput>=1.7.0
opencv-python>=4.8.0
mss>=9.0.0
Pillow>=10.0.0
numpy>=1.24.0
```

### 打包为 EXE

项目已预配置 PyInstaller 打包脚本。一键打包：

```bash
pyinstaller --onefile --windowed ^
    --name "EyeClick_v2.0" ^
    --icon=favicon.ico ^
    --add-data "templates;templates" ^
    --hidden-import=cv2 ^
    --hidden-import=mss ^
    --hidden-import=PIL ^
    --hidden-import=numpy ^
    --hidden-import=pynput ^
    main.py
```

打包后的 EXE 位于 `dist/EyeClick_v2.0.exe`，约 72MB。支持独立运行，无需 Python 环境。

## 快速上手

### 基本流程

```
1. 选择目标窗口（可选） → 2. 添加指令 → 3. 配置参数 → 4. 开始执行
```

1. **选择目标窗口**：在"搜索区域设置"区域点击"刷新"，从下拉列表中选择要操作的窗口，点击"选择目标窗口"完成绑定
2. **添加指令**：点击"添加指令 ▼"按钮，选择"左键点击"、"右键点击"或"等待"
3. **配置指令参数**：
   - 选择模板图片（点击指令必需）
   - 设置置信度阈值
   - 配置倒计时、偏移量、重试次数等参数
4. **测试匹配**（可选）：选中指令后点击"测试匹配"查看匹配效果
5. **开始执行**：点击"开始执行 ▶"运行所有指令

### 指令类型

| 类型 | 说明 | 必需参数 |
|------|------|---------|
| **左键点击** | 匹配模板后模拟鼠标左键点击 | 模板图片、置信度阈值 |
| **右键点击** | 匹配模板后模拟鼠标右键点击 | 模板图片、置信度阈值 |
| **等待** | 等待指定时长，不做任何操作 | 等待时长 |

## 详细配置说明

### 搜索区域

- **不设置（默认）**：在全屏范围内搜索模板，返回最佳匹配位置
- **设置搜索区域**：仅在指定区域内搜索，提高匹配速度，避免误匹配
- **窗口绑定 + 搜索区域**：搜索区域自动转换为相对目标窗口客户区的坐标，适用于窗口内容会移动的场景

设置方式：在添加/编辑指令对话框中，点击"框选识别区域"，弹出全屏截图窗口进行框选。

### 置信度阈值

- **范围**：0.0 ~ 1.0，推荐值 0.7 ~ 0.95
- **值越高**：匹配越精确，但可能遗漏目标
- **值越低**：更容易匹配到目标，但可能误匹配
- **建议**：对于清晰、唯一的图标可设 0.9+；对于可能部分遮挡的场景可设 0.7 ~ 0.8

### 匹配缩放

- **0（自动多尺度）**：默认模式，依次尝试 0.8x、0.9x、1.0x、1.1x、1.2x 五种缩放比例
- **指定值（如 1.0）**：固定缩放比例，仅在特定尺度下搜索
- 若模板在原屏幕上的大小已知，固定缩放可提高匹配速度

### 点击偏移量

匹配位置是模板中心点，点击偏移允许你在此基础上前移/后移：
- **正 X**：向右偏移
- **负 X**：向左偏移
- **正 Y**：向下偏移
- **负 Y**：向上偏移

示例：若模板是某个按钮的图标，但需要点击按钮上的文字，可通过偏移量精确定位。

### 重试机制

- **最大重试次数**：匹配失败后的重试次数（包括首次尝试，最大重试次数为 1 表示一共尝试 2 次）
- **重试间隔**：每次重试之间的等待时间（秒）
- 适用场景：目标出现有延迟（如页面加载、动画播放等）

## 技术架构

### 匹配流程

```
截取屏幕 ──→ 加载模板 ──→ 灰度转换 ──→ 多尺度缩放 ──→ 模板匹配 ──→ 返回最佳位置
   ↑                          ↓
截取区域       若设搜索区域则截取对应范围
（全屏）      ＋窗口偏移转换为屏幕绝对坐标
```

### 坐标系统

| 模式 | 截图方式 | 匹配坐标含义 | 点击坐标计算 |
|------|---------|-------------|-------------|
| 未设搜索区域 | 全屏截图 | 屏幕绝对坐标 | `匹配坐标 + 偏移量` |
| 设搜索区域 | 区域截图 | 相对于区域左上角 | `匹配坐标 + 区域偏移 + 偏移量` |
| 搜索区域+窗口绑定 | 区域截图（窗口偏移后） | 相对于截取区域左上角 | `匹配坐标 + 区域偏移 + 偏移量` |

### DPI 感知

程序启动时自动启用 DPI 感知（`SetProcessDPIAware`），确保在高 DPI 显示环境下屏幕截图和鼠标坐标的一致性。

## 常见问题

### 测试匹配成功，但执行时点击位置错误

**原因**：这是早期的已知 Bug，已在 v2.0 中修复。若你使用的是旧版本，请更新到最新版。

### 匹配总是不成功

1. 检查模板图片是否清晰，建议使用目标在屏幕上实际显示时的截图作为模板
2. 降低置信度阈值（如从 0.9 降至 0.8）
3. 尝试设置搜索区域，排除干扰区域
4. 检查是否开启了多尺度匹配（`match_scale=0`）

### 程序无法定位窗口

- 确保目标窗口未被最小化
- 某些以管理员权限运行的窗口需要本程序也以管理员权限运行
- 绑定窗口后若窗口位置发生移动，程序会自动获取最新位置

### EXE 文件被杀毒软件误报

PyInstaller 打包的 Python 程序常被误报。可将 EXE 添加至杀毒软件白名单，或选择从源码运行。
