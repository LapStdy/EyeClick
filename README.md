# EyeClick

> 基于图像识别的鼠标模拟点击工具

![版本](https://img.shields.io/badge/版本-2.1-blue)
![Python](https://img.shields.io/badge/Python-3.14-green)
![许可证](https://img.shields.io/badge/许可证-MIT-green)
![平台](https://img.shields.io/badge/平台-Windows-lightgrey)

EyeClick 是一款基于计算机视觉的自动化鼠标点击工具。它通过 OpenCV 模板匹配技术识别屏幕上的图像目标，并模拟鼠标点击操作。适用于游戏自动化、软件测试、UI 自动化、重复操作替代等场景。

> 💡 **第一次使用？请先阅读 [完整使用说明书](USER_MANUAL.md)**，内含面向零基础用户的详细图文教程。

## 功能特性

### 核心功能

| 功能 | 说明 |
|------|------|
| **图像模板匹配** | 基于 OpenCV `TM_CCOEFF_NORMED` 算法，支持多尺度缩放匹配 |
| **多尺度搜索** | 默认在 0.8x ~ 1.2x 五种尺度下搜索，也可固定单一尺度 |
| **搜索区域限定** | 支持限定矩形搜索区域，提高匹配效率与准确性 |
| **窗口绑定** | 绑定目标窗口后，搜索区域自动跟随窗口客户区坐标系 |
| **点击偏移量** | 支持水平和垂直方向偏移，精确定位模板内部特定位置 |
| **重试机制** | 匹配失败后自动重试，可配置重试次数与间隔 |
| **置信度阈值** | 可调节匹配阈值（0.0 ~ 1.0），平衡精度与召回率 |
| **DPI 感知** | 启动时自动启用 DPI 感知，确保高 DPI 下坐标一致性 |

### 指令系统

- 支持三种指令类型：**左键点击**、**右键点击**、**等待**
- 指令列表支持添加、编辑、删除、排序、清空
- 每条指令独立配置参数（置信度、偏移量、重试等）
- 配置自动保存至 `config.json`

### 任务执行

- 按序执行多条指令，支持倒计时准备
- 支持启动后自动执行、执行前最小化窗口、执行后自动关闭
- 可选择窗口绑定状态自动校验：窗口未打开时禁止自动执行
- 绑定状态持久化：即使目标窗口未打开，配置信息不丢失

### 测试与调试

- **测试匹配**：执行前对单条指令进行匹配预览，显示匹配位置、置信度和可视化标记
- 实时日志输出，记录执行过程与结果
- 状态指示器与进度显示

### 用户界面

- 基于 tkinter 的桌面图形界面
- 模板图片预览
- 匹配结果可视化（绿色方框 + 红色中心点）

## 安装

### 下载预编译版本（推荐）

从 GitHub Releases 页面直接下载已打包好的 EXE 文件，无需安装 Python，下载后双击即可运行：

[**下载 EyeClick 最新版本**](https://github.com/LapStdy/EyeClick/releases)

### 从源码运行

- Python 3.8+
- Windows 7 及以上

```bash
git clone https://github.com/LapStdy/EyeClick.git
cd EyeClick
pip install -r requirements.txt
python main.py
```

### 依赖清单

```
pynput>=1.7.0
opencv-python>=4.8.0
mss>=9.0.0
Pillow>=10.0.0
numpy>=1.24.0
```

### 自行打包为 EXE

如需自行打包，可使用 PyInstaller：

```bash
pip install pyinstaller
pyinstaller --onefile --windowed ^
    --name "EyeClick_v2.0" ^
    --icon=favicon.ico ^
    --add-data "templates;templates" ^
    --hidden-import=cv2 ^
    --hidden-import=mss ^
    --hidden-import=PIL ^
    --hidden-import=numpy ^
    --hidden-import=pynput ^
    main.py
```

打包产物位于 `dist/EyeClick_v2.0.exe`，约 72MB。

## 快速开始

### 基本流程

```
添加指令 → 选择模板 → 配置参数 → 测试匹配（可选） → 开始执行
```

1. **添加指令**：点击"添加指令 ▼"按钮，选择"左键点击"或"右键点击"
2. **选择模板图片**：在弹出对话框中点击"加载模板文件"，选择提前截取的目标图片
3. **配置参数**：设置置信度阈值、倒计时、偏移量、重试次数等
4. **测试匹配**（推荐）：选中指令，点击"测试匹配"预览匹配效果
5. **开始执行**：确认无误后点击"开始执行 ▶"

> 详细操作步骤请参考 [完整使用说明书](USER_MANUAL.md#4-完整使用流程图文步骤)。

### 指令类型

| 类型 | 用途 | 必需参数 |
|------|------|---------|
| **左键点击** | 匹配模板后模拟鼠标左键点击 | 模板图片、置信度阈值 |
| **右键点击** | 匹配模板后模拟鼠标右键点击 | 模板图片、置信度阈值 |
| **等待** | 等待指定时长，不做任何操作 | 等待时长 |

## 技术架构

### 匹配流程

```
截取屏幕 ──→ 灰度转换 ──→ 多尺度缩放 ──→ 模板匹配 ──→ 返回最佳位置
    │                                            (TM_CCOEFF_NORMED)
    ├── 未设搜索区域：全屏截图 → 返回屏幕绝对坐标
    └── 设搜索区域  ：区域截图 → 返回区域相对坐标 + 区域偏移量
```

### 坐标系统

| 模式 | 截图方式 | 匹配坐标 | 点击坐标计算 |
|------|---------|---------|-------------|
| 未设搜索区域 | 全屏截图 | 屏幕绝对坐标 | `匹配坐标 + 偏移量` |
| 搜索区域 | 区域截图 | 区域相对坐标 | `匹配坐标 + 区域偏移 + 偏移量` |
| 搜索区域 + 窗口绑定 | 窗口偏移后区域截图 | 区域相对坐标 | `匹配坐标 + 区域偏移 + 偏移量` |

### DPI 感知

程序启动时通过 `SetProcessDPIAware` 启用 DPI 感知，确保高 DPI 显示环境下截图坐标与鼠标坐标的一致性。

## 常见问题

### 匹配总是不成功

1. 确保模板图片为目标在屏幕上的实际截图，未经缩放或滤镜处理
2. 适当降低置信度阈值（如从 0.90 降至 0.80）
3. 设置搜索区域，排除干扰区域
4. 确认多尺度匹配已开启（`match_scale=0`）

### 测试匹配成功，但执行时点击位置错误

请确认使用 v2.0 及以上版本。旧版本存在搜索区域坐标未转换的 Bug，已在新版中修复。

### 程序无法定位目标窗口

- 确保目标窗口已打开且未被最小化
- 若目标程序以管理员权限运行，本程序也需以管理员权限运行
- 绑定窗口后若窗口移动，程序会自动获取最新位置

### 窗口绑定状态丢失

v2.1 起已修复：即使目标窗口未打开，绑定信息会持久化保存，重新启动后显示橙色提示"已绑定: xxx（窗口未打开）"。

### EXE 被杀毒软件误报

PyInstaller 打包的 Python 程序常见于杀毒软件误报。可将 EXE 添加至杀毒软件白名单，或从源码运行。
