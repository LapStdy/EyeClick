# EyeClick 使用说明书

## 写给第一次使用的你

欢迎使用 EyeClick！这是一款"看图识位置、自动帮你点鼠标"的工具。

如果你经常需要重复做这样一个操作：**打开某个软件 → 等待画面出现 → 点击某个按钮**，那么 EyeClick 可以帮你自动完成这些步骤。你只需要提前告诉它"要点击什么东西长什么样"，它就能自己在屏幕上找到并点击。

**举个例子：**
- ✅ 每天早上自动登录办公系统
- ✅ 玩游戏时自动点击某个刷新按钮
- ✅ 批量处理文件时自动点击"确定"弹窗
- ✅ 软件测试中重复点击相同的界面元素

## 1. 如何获取程序

### 方式一：下载已发布的版本（推荐，最简单）

访问 GitHub Releases 页面下载预编译好的 EXE 文件：

[**https://github.com/LapStdy/EyeClick/releases**](https://github.com/LapStdy/EyeClick/releases)

1. 打开上面的链接，你会看到所有已发布的版本
2. 找到最新的版本（最上面那个）
3. 在版本信息下方的 **"Assets"** 区域点击 `EyeClick_v2.0.exe` 即可下载
4. 下载完成后双击运行，不需要安装任何东西

建议在电脑上新建一个文件夹（如 `D:\EyeClick`），把 exe 文件放进去再运行。配置文件会自动保存在 exe 旁边。

> 小提示：如果杀毒软件弹出警告，这是因为 Python 打包的程序容易被误报。选择"允许运行"或"添加信任"即可。如果担心安全问题，可以选择方式三从源代码运行。

### 方式二：直接运行 EXE（如果你已经有 exe 文件）

1. 拿到 `EyeClick_v2.0.exe` 文件
2. 双击即可运行，不需要安装任何东西
3. 建议在电脑上新建一个文件夹（如 `D:\EyeClick`），把 exe 文件放进去再运行。配置文件会自动保存在 exe 旁边

> 小提示：如果杀毒软件弹出警告，这是因为 Python 打包的程序容易被误报。选择"允许运行"或"添加信任"即可。

### 方式三：从源代码运行（需要懂一点 Python）

如果你电脑上已经安装了 Python 3.8 或更高版本：

```
1. 下载整个项目文件夹到电脑上
2. 打开"命令提示符"或"PowerShell"
3. 进入项目文件夹，执行：pip install -r requirements.txt
4. 执行：python main.py
```

│  序号 │ 类型 │ 倒计时 │ 模板预览 │ 置信度 │ 偏移    │

### 各区域的作用

| 区域 | 干什么用的 |
|------|-----------|
| **① 窗口绑定区** | 可选。如果你的目标在某个特定软件窗口里，可以在这里选中那个窗口。这样即使窗口移动了，程序也能找到 |
| **② 指令列表** | 这里列出所有你设置好的操作步骤（比如：先点A → 等2秒 → 再点B） |
| **③ 操作按钮** | 添加新指令、修改、排序等 |
| **④ 控制选项** | 控制程序执行时的行为（是否最小化窗口、是否完成后关闭等） |
| **⑤ 执行日志** | 显示程序正在做什么、有没有找到目标 |
| **⑥ 状态区** | 显示当前状态、进度 |

## 3. 核心概念速成

### 什么是"模板"？

模板就是**你要让程序去找的那张图片**。比如你想让程序点击一个"确定"按钮，那你就需要先截一张"确定"按钮的图片作为模板。

**怎么做模板？** 用 Windows 自带的截图工具（Win+Shift+S）截取目标区域，保存为 PNG 图片，然后在 EyeClick 中加载这张图片。

### 什么是"置信度"？

置信度就是**程序有多大把握认为自己找到了**。
- **1.00（100%）**：完全一模一样
- **0.80（80%）**：有八成像

一般设到 **0.80~0.90** 之间比较合适。设太高了可能找不到，设太低了可能找错。

### 什么是"搜索区域"？

搜索区域就是**告诉程序别在整屏找，只在这个小范围里找**。这样可以：
- 找得更快
- 不容易找错（排除了干扰区域）

## 4. 完整使用流程（图文步骤）

下面以"每天自动登录某款软件"为例，演示完整的使用流程。

### 准备工作

打开你想要操作的目标软件，让它停留在你想要开始操作的界面。

### 第一步：添加指令

点击 **"添加指令 ▼"** 按钮，选择 **"左键点击"**。

![操作示意：点击添加指令按钮，选择左键点击]

此时会弹出一个"添加指令"窗口。

### 第二步：选择模板图片

在弹出窗口中：

1. 点击 **"加载模板文件"** 按钮
2. 在弹出的文件选择窗口中，找到你提前截好的目标图片（比如"登录按钮"的截图）
3. 选中图片，点击"打开"

加载成功后，你会看到模板图片显示在预览区域中，下方显示文件名。

### 第三步：设置参数（重要！）

在同一个窗口中，你需要设置以下参数：

#### 倒计时（执行前等待）
- **这是什么**：在执行这条指令前等待多少秒
- **设为**：`2.0` 秒
- **为什么**：给程序一点时间切换到目标界面

#### 置信度阈值
- **这是什么**：匹配的严格程度
- **建议**：先设为 `0.85`
- **如果找不到**：慢慢调低到 `0.80`、`0.75`
- **如果找错了**：调高到 `0.90`、`0.95`

#### 点击偏移量
- **这是什么**：找到目标后，在目标的基础上再偏移多少像素点击
- **通常**：保持 `0` 和 `0` 即可（直接在目标中心点击）
- **特殊场景**：比如你截了一张图标，但想点图标旁边的文字，可以设置偏移

#### 最大重试次数
- **这是什么**：如果一次没找到，再试几次
- **建议**：设为 `2` ~ `3` 次
- **间隔**：每次重试之间等 `0.5` ~ `1.0` 秒

#### 匹配缩放
- **这是什么**：如果目标在屏幕上的大小和模板不一样，程序自动缩放匹配
- **建议**：保持 `0`（自动多尺度），这样大一点小一点都能找到

### 第四步：框选搜索区域（可选但推荐）

在同一个窗口中，找到"识别区域"部分：

1. 点击 **"框选识别区域"** 按钮
2. 程序会截取当前全屏画面并弹出一个新窗口
3. 在截图上**按住鼠标左键拖拽**，框出你想要搜索的范围（不用太大，刚好包住目标即可）
4. 松开鼠标，区域就选好了

设置成功后，你会看到类似这样的提示：
`区域: (100, 200) → (300, 280)  (200×80)`

### 第五步：确认并保存

点击 **"确定"** 按钮，关闭添加窗口。你会看到指令列表中出现了刚才添加的指令。

### 第六步：测试匹配（强烈建议先做这一步！）

1. 在指令列表中**点击选中**刚才添加的指令（点一下让它变蓝）
2. 点击 **"测试匹配"** 按钮
3. 程序会进行匹配测试，并弹出一个结果窗口

**测试结果解读：**

| 结果 | 含义 | 怎么办 |
|------|------|--------|
| 匹配成功，位置(800, 450) 置信度95.00% | ✅ 找到了！位置正确 | 可以放心执行 |
| 匹配失败，未找到匹配结果 | ❌ 程序找不到目标 | 调低置信度，或检查模板图片 |
| 匹配成功，但位置不对 | ⚠️ 找错了地方 | 调高置信度，或设置搜索区域 |

在结果窗口中，你会看到：
- 一张截图，**绿色方框**标出了找到的位置
- **红色圆点**标出了中心点
- 左上角显示坐标信息

### 第七步：添加更多指令

重复第一步到第六步，添加所有需要的操作。例如：

| 序号 | 类型 | 模板 | 说明 |
|------|------|------|------|
| 1 | 左键点击 | 启动图标 | 打开软件 |
| 2 | 等待 | — | 等5秒让软件加载 |
| 3 | 左键点击 | 登录按钮 | 点击登录 |
| 4 | 等待 | — | 等3秒 |
| 5 | 左键点击 | 确定按钮 | 点击确定弹窗 |

> 点击"添加指令 ▼" → "等待（无操作）"可以添加等待指令。

### 第八步：开始执行

1. 确认所有指令已经按顺序排好
2. 点击 **"开始执行 ▶"** 按钮
3. 程序会按顺序一条一条执行你的指令

**执行过程中：**
- 状态区会显示"执行中"
- 日志区会显示每一步的详细信息
- 如果某条指令匹配失败，程序会按照你设置的重试次数自动重试

**如何停止？**
- 点击 **"停止执行 ■"** 按钮可以随时终止执行

## 5. 详细功能说明

### 5.1 指令列表操作

#### 添加指令

点击 **"添加指令 ▼"**，有三种类型可选：

| 指令类型 | 何时使用 |
|---------|---------|
| **左键点击** | 需要点击某个按钮、图标、链接时 |
| **右键点击** | 需要弹出右键菜单时 |
| **等待（无操作）** | 需要等待某个画面加载出来时 |

#### 编辑指令

1. 在列表中**双击**某条指令，或选中后点击 **"编辑"** 按钮
2. 在弹出的窗口中修改参数
3. 点击"确定"保存修改

#### 调整顺序

用 **"上移"** 和 **"下移"** 按钮调整指令的执行顺序。

#### 删除指令

选中指令后点击 **"删除"** 按钮。也可以点击 **"清空"** 一次删除所有指令。

### 5.2 窗口绑定

#### 什么情况下需要绑定窗口？
- 你的目标程序窗口可能会被移动
- 你的搜索区域是相对于窗口内部设置的
- 你想在窗口最小化时也能操作（注意：部分窗口最小化后无法截图）

#### 如何绑定窗口？
1. 先打开你的目标软件
2. 在 EyeClick 中点击 **"刷新"** 按钮获取当前所有窗口列表
3. 从下拉列表中找到你的窗口名称（通常会显示窗口标题）
4. 点击 **"选择目标窗口"**
5. 如果绑定成功，"当前未绑定窗口"会变成绿色文字显示窗口名

#### 取消绑定

目前不支持直接取消绑定。如果需要解绑，可以关闭程序并手动编辑 `config.json` 文件，将 `target_window_title` 改为空。或者重启程序后选择另一个窗口。

### 5.3 控制选项

#### 高亮显示匹配位置
- **勾选后**：执行时会用特殊效果标出找到的位置（如果程序的"高亮"回调已实现）
- **建议**：保持勾选，方便调试

#### 执行前最小化本窗口
- **勾选后**：点击"开始执行"后，EyeClick 窗口会自动最小化到任务栏
- **为什么需要**：避免 EyeClick 的窗口挡住目标界面

#### 完成后自动关闭
- **勾选后**：所有指令执行完毕后，程序会自动关闭（5秒倒计时后）
- **适用于**：无人值守自动运行

#### 启动后自动执行
- **勾选后**：下次启动 EyeClick 时会自动开始执行
- **适用于**：设置好后每次开机自动运行的场景

### 5.4 全局置信度阈值

在窗口绑定区下方的滑块可以设置**全局默认置信度**。当你添加新指令时，会使用这个值作为默认值。已经添加的指令不受影响。

### 5.5 配置保存

点击 **"保存设置"** 可以手动保存当前配置。程序关闭时也会自动保存。

配置保存在和 EXE 同目录的 `config.json` 文件中，包括：
- 所有指令列表
- 绑定的窗口信息
- 控制选项的设置

## 6. 常见问题与解决办法

### Q1：测试匹配成功了，但执行时点错了位置

**可能原因：** 之前版本的 Bug。请确保你使用的是最新版本（v2.0+）。

**第二个可能：** 你在执行过程中移动了目标窗口，但你没有绑定窗口。解决方案：绑定目标窗口。

### Q2：程序总是说"匹配失败"

请按以下顺序排查：

1. **检查模板图片**
   - 模板必须是目标在屏幕上实际显示的样子（颜色、大小尽量一致）
   - 建议用截图工具截取，不要用 PS 处理过的

2. **降低置信度阈值**
   - 从 0.90 降到 0.85，再降到 0.80
   - 每次降低 0.05，然后测试匹配

3. **设置搜索区域**
   - 在目标附近框一个小范围，排除干扰

4. **检查模板路径**
   - 如果是 EXE 版本，模板图片要放在你知道的地方（如桌面）
   - 不要在运行后移动或删除模板图片

5. **确认目标在屏幕上可见**
   - 如果目标在另一个被遮挡的窗口里，程序看不到它

### Q3：程序找到的位置不对，点到了错误的地方

1. **提高置信度阈值**（从 0.80 提高到 0.90）
2. **设置搜索区域**，缩小搜索范围
3. **更换模板图片**，使用更有特征的部分作为模板

### Q4：窗口列表里找不到我的目标窗口

1. 点击 **"刷新"** 按钮重新获取列表
2. 确保目标窗口已经打开并且没有被最小化
3. 某些系统窗口或没有标题的窗口可能不显示
4. **管理员权限问题**：如果目标程序是以管理员身份运行的，EyeClick 也需要以管理员身份运行（右键 exe → 以管理员身份运行）

### Q5：点击"开始执行"后好像没反应

1. 检查是否设置了倒计时（默认 2 秒），程序可能在倒计时中
2. 查看日志区是否有提示信息
3. 点击 **"停止执行"** 后重新试

### Q6：如何让程序每天自动运行？

1. 设置好所有指令和控制选项
2. 勾选 **"启动后自动执行"**
3. 勾选 **"完成后自动关闭"**
4. 保存设置，关闭程序
5. 使用 Windows 的"任务计划程序"创建一个任务：
   - 触发器：每天指定时间
   - 操作：启动 `EyeClick_v2.0.exe`

### Q7：EXE 文件被杀毒软件删除了

1. 在杀毒软件中将 exe 文件添加为信任/白名单
2. 或者从源代码运行（方式二）

## 7. 使用技巧与建议

### 技巧一：截图模板的技巧

好的模板 = 成功率高的关键。

```
✅ 截取有特征的局部（如按钮上的文字图标）
✅ 截取时尽量和实际显示大小一致
❌ 不要截取大片重复背景（如纯色区域）
❌ 不要截取会变化的元素（如加载动画）
```

**最佳实践：** 在目标界面上，用截图工具框选目标按钮或图标，保存为 PNG 文件。

### 技巧二：使用搜索区域

搜索区域是提高匹配成功率最有效的方法之一。

- **何时设置**：当屏幕上有多个相似元素时
- **如何设置**：在添加指令时点击"框选识别区域"，在目标周围框一个稍大一点的范围
- **多小合适**：刚好包住目标即可，留出 10~20 像素的余量

### 技巧三：等待指令的合理使用

在实际自动化中，**等待**是最容易被忽略但最重要的指令。

| 场景 | 建议等待时间 |
|------|------------|
| 点击后界面立即切换 | 1~2 秒 |
| 网络加载较慢 | 3~5 秒 |
| 启动大型软件 | 5~10 秒 |

### 技巧四：重试机制的使用

对于加载时间不确定的场景（如网络请求），建议：
- 重试次数：3~5 次
- 重试间隔：1~2 秒

### 技巧五：测试匹配的习惯

**每次添加或修改指令后，都先点一下"测试匹配"**。这是最推荐的调试习惯，可以避免执行时才发现问题。

### 技巧六：多指令的顺序编排

一个好的指令序列应该是这样的：

```
指令1: 左键点击 → 打开程序图标（倒计时 2s）
指令2: 等待     → 等 5 秒让程序加载
指令3: 左键点击 → 点击登录框（倒计时 1s）
指令4: 等待     → 等 2 秒
指令5: 左键点击 → 点击登录按钮（倒计时 1s）
```

## 附录：配置文件 (config.json) 说明

`config.json` 是程序的配置文件，保存在和 `EyeClick_v2.0.exe` **同一个文件夹**中。程序启动时会自动读取，关闭时自动保存。你也可以用记事本手动编辑它。

> **如何打开？** 右键 `config.json` → 选择"打开方式" → 选择"记事本"

下面逐一说明每个参数的作用。

### 顶层参数

以下参数直接写在 `config.json` 的根层级（打开文件就能看到）。

#### `global_confidence`
- **作用**：全局默认置信度阈值。添加新指令时，自动使用这个值作为指令的置信度
- **是否可用**：是 ✅
- **如何修改**：
  - 在程序界面拖动"全局置信度阈值"滑块，点击"保存设置"
  - 或直接修改数值：`0.85` 表示 85%

#### `search_mode`
- **作用**：搜索模式（保留字段）
- **是否可用**：否 ❌（暂未使用，当前始终为 `"fullscreen"`）
- **建议**：保持默认，不要修改

#### `target_window_title`
- **作用**：绑定的目标窗口标题。设置了之后，程序启动时会自动恢复窗口绑定
- **是否可用**：是 ✅
- **如何修改**：
  - 在程序界面点击"选择目标窗口"完成绑定，会自动保存
  - 如果想取消绑定，可以手动打开 `config.json`，把这行改为：`"target_window_title": null`
- **注意事项**：如果窗口标题包含中文，记事本编辑后保存时要确保编码为 UTF-8

#### `auto_close`
- **作用**：所有指令执行完毕后，是否自动关闭程序
- **是否可用**：是 ✅
- **如何修改**：
  - 勾选/取消"完成后自动关闭"复选框，点击"保存设置"
  - 或手动修改：`true` = 自动关闭，`false` = 不关闭

#### `auto_start`
- **作用**：程序启动后是否自动开始执行所有指令
- **是否可用**：是 ✅
- **如何修改**：
  - 勾选/取消"启动后自动执行"复选框，点击"保存设置"
  - 或手动修改：`true` = 自动执行，`false` = 不执行
- **注意事项**：如果绑定的窗口未打开，即使设为 `true` 也不会自动执行

#### `minimize_on_execute`
- **作用**：点击"开始执行"后，是否自动将 EyeClick 窗口最小化到任务栏
- **是否可用**：是 ✅
- **如何修改**：
  - 勾选/取消"执行前最小化本窗口"复选框，点击"保存设置"
  - 或手动修改：`true` = 最小化，`false` = 保持窗口

#### `highlight_match`
- **作用**：执行过程中是否高亮显示匹配到的位置
- **是否可用**：是 ✅
- **如何修改**：
  - 勾选/取消"高亮显示匹配位置"复选框，点击"保存设置"
  - 或手动修改：`true` = 高亮，`false` = 不高亮

#### `log_level`
- **作用**：日志输出级别（保留字段）
- **是否可用**：否 ❌（暂未使用）
- **建议**：保持默认 `"info"`，不要修改

#### `task_id_counter`
- **作用**：记录下一个新指令的编号。每添加一条指令，这个数字会自动加 1
- **是否可用**：是 ✅（程序自动管理，不需要手动修改）
- **建议**：**不要手动修改**，让程序自己管理就好

### 指令列表（tasks）

`tasks` 是一个数组（用方括号 `[]` 括起来），里面每一条 `{}` 代表一个指令。每个指令包含以下参数：

#### `id`
- **作用**：指令序号（从 1 开始）
- **是否可用**：是 ✅
- **如何修改**：在程序中上移/下移指令时，序号会自动重新排列。不建议手动修改

#### `task_type`
- **作用**：指令类型
- **可选值**：`"left_click"`（左键点击）、`"right_click"`（右键点击）、`"wait"`（等待）
- **是否可用**：是 ✅
- **如何修改**：在添加/编辑指令对话框中修改"指令类型"下拉框

#### `countdown`
- **作用**：执行这条指令前等待的秒数
- **范围**：0 ~ 3600
- **是否可用**：是 ✅
- **如何修改**：在添加/编辑指令对话框中修改"倒计时"输入框

#### `template_path`
- **作用**：模板图片的文件路径。可以是绝对路径（如 `D:\图片\按钮.png`），也可以是相对于程序所在文件夹的相对路径
- **是否可用**：是 ✅（仅点击类指令）
- **如何修改**：在添加/编辑指令对话框中点击"加载模板文件"选择图片
- **注意事项**：如果移动或删除了模板图片，程序将无法匹配

#### `confidence_threshold`
- **作用**：这条指令的匹配置信度阈值
- **范围**：0.0 ~ 1.0
- **是否可用**：是 ✅
- **如何修改**：在添加/编辑指令对话框中拖动"置信度阈值"滑块
- **说明**：每条指令可以有自己的置信度，不受全局设置影响

#### `match_scale`
- **作用**：匹配缩放比例
- **范围**：0 ~ 5（0 表示自动多尺度匹配）
- **是否可用**：是 ✅
- **如何修改**：在添加/编辑指令对话框中修改"匹配缩放"输入框

#### `click_offset_x`
- **作用**：匹配到目标后，在水平方向上偏移多少像素再点击（正数向右，负数向左）
- **范围**：-5000 ~ 5000
- **是否可用**：是 ✅（仅点击类指令）
- **如何修改**：在添加/编辑指令对话框中修改"点击偏移 X"输入框

#### `click_offset_y`
- **作用**：匹配到目标后，在垂直方向上偏移多少像素再点击（正数向下，负数向上）
- **范围**：-5000 ~ 5000
- **是否可用**：是 ✅（仅点击类指令）
- **如何修改**：在添加/编辑指令对话框中修改"点击偏移 Y"输入框

#### `search_region`
- **作用**：限定搜索区域。格式为 `[left, top, right, bottom]` 四个数字，分别表示区域左上角和右下角的坐标
- **示例**：`[100, 200, 300, 280]` 表示从屏幕坐标 (100, 200) 到 (300, 280) 的矩形区域
- **是否可用**：是 ✅
- **如何修改**：
  - 在添加/编辑指令对话框中点击"框选识别区域"，用鼠标拖拽选择
  - 或手动编辑：`null` 表示不限制（全屏搜索）；`[左, 上, 右, 下]` 表示限定区域
- **注意事项**：如果绑定了目标窗口，这些坐标会自动转换为相对于窗口客户区的位置

#### `wait_duration`
- **作用**：等待指令的持续时间（秒）
- **范围**：0 ~ 86400
- **是否可用**：是 ✅（仅等待类指令）
- **如何修改**：在添加/编辑指令对话框中修改"等待时长"输入框

#### `max_retries`
- **作用**：匹配失败后最多重试几次。例如设为 2，表示首次匹配 + 2 次重试，一共尝试 3 次
- **范围**：0 ~ 100
- **是否可用**：是 ✅
- **如何修改**：在添加/编辑指令对话框中修改"最大重试"输入框

#### `retry_interval`
- **作用**：每次重试之间等待的秒数
- **范围**：0 ~ 60
- **是否可用**：是 ✅
- **如何修改**：在添加/编辑指令对话框中修改"间隔"输入框

### 手动编辑配置文件的安全须知

1. **先关闭程序再编辑**：程序运行时可能会覆盖你的手动修改
2. **注意 JSON 格式**：逗号、冒号、引号都不能少，也不能多。如果不确定，可以先备份一份 `config.json`
3. **UTF-8 编码**：保存时一定要选择 UTF-8 编码（记事本默认就是）
4. **错误的后果**：如果格式错误，程序启动时无法读取配置，会自动创建一个新的空白配置文件（旧的配置会丢失）
